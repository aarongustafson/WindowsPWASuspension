Testing Suspension in Windows PWA

This page will run an expensive, battery draining operation using setInterval (which wakes the CPU) and fetch (which will wake up the radio) in an attempt to burndown the battery of a mobile device. Most browsers should suspend this tab in the background, but we want to make sure the WWAHost on Windows Phones is correctly doing that as well. An additional event listener is in place to track suspensions triggered from WinRT.
